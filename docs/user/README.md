# 首页

## 搜索接口

**功能：**
用户可以通过起始地址|目的地|出发时间|结束时间|车牌号 等条件来筛选获取想要的申请记录。

- 提供参数：起始地址|目的地|出发时间|结束时间|车牌号 其中某一个
- 返回结果：按照时间排序的数组


## 轮播图接口

**功能：**
PC端可以通过后台来维护这个接口需要展示的内容，用来通知用户某些信息等.比如：五一休假不提供车辆等等。
- 提供参数：无
- 返回参数：图片base64码以及必要的文字内容

::: warning
待定
:::

## 展示用户审批中的接口

**功能：**
展示用户正在申请中的用车申请，这里需要按时间排序，并且只取前面几个，具体个数有app端传给后台。

- 提供参数：用户id，审批状态，数据个数
- 返回参数：申请时间、申请汽车图片base64码、起始时间（地点）、结束时间（地点）、申请状态

## 展示用户已经完成车辆申请的接口

**功能：**
展示用户已经结束的用车申请，这里需要按时间排序，并且只取前面几个，具体个数有app端传给后台。

- 提供参数：用户id，审批状态，数据个数
- 返回参数：申请时间、申请汽车图片base64码、起始时间（地点）、结束时间（地点）、申请状态

## 用户撤销用车申请的接口

**功能：**
用户可以通过点击撤销按钮，撤销本次用车申请。

- 提供参数：用户id、当前申请流程id
- 返回参数：是否撤销成功

::: danger
删除
:::


## 用户催办申请流程的接口

**功能：**
用户可以通过点击催办按钮，提示当前正在审批的用户。（这里通过推送一条消息给当前的审批人）

- 提供参数：用户id、当前申请流程id
- 返回参数：返回成功与否
