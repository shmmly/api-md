# 车辆申请

## 车辆类型选择接口

**功能：**
提供可以选择的车辆类型，这里有2中选择
- 前端写死车辆类型
- 后端返回车辆类型，供用户来选择。
****
- 提供参数：无
- 返回参数：车辆的类型以及车辆类型对应图片的base64码。

::: danger danger
静态写死，具体类型参照前端
:::

## 事由接口

**功能：**
用户申请车辆的时候，需要填写申请原因，这个原因可能有很多次是重复的。
这个接口返回用户使用频次最多的、理由。

- 提供参数：用户id
- 返回参数：一个数组、

::: danger danger
事由前端写死
:::


### 事由设定接口
**功能：**
用户可以自己设定几个常用的事由。

- 提供参数：用户id，设定的事由
- 返回参数：是否设定成功
::: warning
待后续版本优化
:::

## 表单提交接口
**功能：**
用户点击保存，提交本次的申请

- 提供参数：用户id、出发地（时间）、目的地（时间）、车辆类型、用车人、用车人数、联系人、联系电话、事由
- 返回参数：是否提供成功
::: tip
提供两个接口，一个新增修改，一个提交申请
先进行新增业务，新增完成之后提交申请
:::
